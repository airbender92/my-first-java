这4个主题（多线程、网络编程、Lambda/Stream、MySQL + JDBC）正是Java后端开发的“四大支柱”。它们分别解决的是：**性能（并发）**、**通信（网络）**、**代码简洁（函数式）** 和 **存储（数据库）**。

建议你按照 **“由浅入深、由工具到思想”** 的顺序推进。我为你设计了一个**为期4~6周的沉浸式路线图**，以最终能写出一个 **“支持多人在线的聊天室+消息存储”** 项目为终点。

### 🗺️ 第一阶段：代码美学与效率（第1周）—— Lambda 与 Stream API

这是通往现代Java（以及未来学习Spark、Flink）的钥匙。它让你从“怎么做”转向“做什么”。

- **学习目标**：用更少的代码完成复杂的数据过滤、排序和聚合。
- **核心知识点**：
    - **函数式接口**：`Consumer`（消费）、`Supplier`（供给）、`Function`（转换）、`Predicate`（判断）。
    - **Stream 三剑客**：**创建**（`stream()`）、**中间操作**（`filter`过滤、`map`映射、`sorted`排序）、**终端操作**（`collect`收集、`forEach`遍历、`reduce`归约）。
    - **Optional**：优雅地解决空指针异常（`NullPointerException`）。
- **实战练习**：创建一个员工列表，用 Stream 筛选出“薪资大于8000且部门为技术部”的员工，按薪资倒序排序，取出前3名的姓名放入新列表。
- **重点提醒**：**不要滥用并行流（`parallelStream`）**，除非你搞清楚了线程安全问题。

### 🗄️ 第二阶段：数据持久化（第2周）—— MySQL 与 JDBC

程序不能断电就丢数据，数据库是项目的“命脉”。先学SQL基础，再学Java如何连它。

- **学习目标**：能独立设计简单的数据表，并用Java代码进行增删改查（CRUD）。
- **核心知识点**：
    - **MySQL基础**：`DDL`（建表）、`DML`（增删改）、`DQL`（单表查询、分组`group by`、排序`order by`）。
    - **JDBC 核心流程**（背熟这5步）：
        1. 加载驱动 → 2. 获取连接（`DriverManager.getConnection`）→ 3. 创建`PreparedStatement`（**防止SQL注入，必须用这个！**）→ 4. 执行查询获得`ResultSet` → 5. 释放资源（`close`）。
    - **事务管理**：理解 `connection.setAutoCommit(false)` + `commit()` + `rollback()`。
- **实战练习**：设计一张 `user` 表（id, username, password），用 JDBC 实现用户的注册（插入）和登录（查询）功能。

### 🌐 第三阶段：网络通信（第3周）—— 网络编程（Socket）

让Java程序能隔着网线“对话”，这是所有互联网应用的基础。

- **学习目标**：理解 TCP 协议，实现客户端（Client）与服务端（Server）的请求-响应模型。
- **核心知识点**：
    - **ServerSocket 与 Socket**：服务端监听端口（如 `new ServerSocket(8888)`），客户端发起连接。
    - **传输通道**：结合前面学的 I/O 流（`BufferedReader` / `PrintWriter`），发送和接收字符串或对象。
    - **短连接 vs 长连接**：理解HTTP请求背后的Socket原理。
- **实战练习**：编写一个简单的“一问一答”程序。客户端发一句“你好”，服务端回复一句“世界你好”，然后断开连接。**注意：** 这个阶段还是单线程的，服务端同时只能处理一个客户端。

### ⚡ 第四阶段：性能飞跃（第4周）—— 多线程与高并发基础

这是区分“码农”和“工程师”的分水岭。核心在于解决“多个人同时用”时的资源冲突。

- **学习目标**：让程序能同时处理多个任务，并保证数据安全。
- **核心知识点**：
    - **创建线程**：继承 `Thread` vs 实现 `Runnable`（更常用），以及 **`Callable` + `Future`**（有返回值）。
    - **线程安全**：理解 `synchronized`（同步锁）和 `Lock`（显式锁）的区别，以及 `volatile` 的可见性。
    - **JUC 并发工具包**（重中之重）：
        - **线程池**：`Executors.newFixedThreadPool(10)`（**千万别用`Executors`创建无界队列，建议用`ThreadPoolExecutor`自定义参数**）。
        - **并发集合**：`ConcurrentHashMap`（面试常客，替代 `Hashtable`）。
- **实战练习**：模拟 100 个人同时抢 10 张票，使用 `AtomicInteger` 或 `synchronized` 保证库存不会超卖。

### 🎯 终极整合项目：多线程群聊系统 + 消息归档

把前四周的知识全串起来，做一个 **“支持私聊和群发的控制台聊天室”**：

1. **服务端（多线程核心）**：主线程负责接收新客户端连接，每来一个客户端，就新建一个线程（或交给线程池）专门处理该客户端的收发消息。
2. **消息广播（网络通信）**：服务端维护一个 `CopyOnWriteArraySet` 存所有在线客户端。当收到一条消息，遍历集合发送给所有人（群发）或指定某人（私聊）。
3. **历史记录（JDBC+MySQL）**：服务端收到每一条消息，异步（用新线程或队列）写入 MySQL 的 `chat_log` 表（字段：id, sender, receiver, content, send_time）。
4. **优雅下线（Lambda+异常）**：客户端断开时，利用 `try-with-resources` 自动关闭流，并从在线集合中移除。

### 📚 学习避坑指南（很重要！）

| 主题 | 常见大坑 | 正确姿势 |
| :--- | :--- | :--- |
| **多线程** | 以为 `synchronized` 锁住代码就万事大吉，结果锁了不同的对象。 | 明确**锁的是同一个对象**（如 `this` 或同一个类对象）。 |
| **JDBC** | 用 `Statement` 拼接 SQL，结果被 SQL 注入炸掉数据库。 | **永远只用 `PreparedStatement`** 传参。 |
| **网络编程** | 服务端读取流时用 `while((line=in.readLine())!=null)` 卡死。 | 注意 `readLine()` 是阻塞的，需要约定结束标记，或者使用 `DataInputStream` 先写长度。 |
| **Lambda** | 在 Lambda 表达式中使用外部非 `final` 变量报错。 | 外部变量必须声明为 `final` 或实际上的 `final`（即不重新赋值）。 |
